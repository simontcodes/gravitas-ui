  ## 1. Install the package

  <div className="gv-doc-panel">
    <p className="gv-doc-section-copy" style={{ marginTop: 0 }}>
      Install Gravitas and Bootstrap in the consuming Angular project:
    </p>

```bash
npm install gravitas-ui bootstrap
```

    <p className="gv-doc-section-copy" style={{ marginBottom: 0 }}>
      Gravitas expects Angular peer dependencies to come from the host application, while Bootstrap
      provides the supported baseline shell environment.
    </p>
  </div>

  ## 2. Import global styles once

  <div className="gv-doc-panel">
    <p className="gv-doc-section-copy" style={{ marginTop: 0 }}>
      Add these imports in your global styles or application entrypoint:
    </p>

```ts
import 'bootstrap/dist/css/bootstrap.min.css';
import 'gravitas-ui/styles/gravitas.css';
```

    <p className="gv-doc-section-copy" style={{ marginBottom: 0 }}>
      `gravitas.css` contains the Gravitas token and theme layers. Do not import these files
      repeatedly in feature modules or component-level styles.
    </p>
  </div>

  ## 3. Set theme and density at the app shell

  <div className="gv-doc-panel">
    <p className="gv-doc-section-copy" style={{ marginTop: 0 }}>
      Gravitas reads theme and density from root attributes:
    </p>

```html
<html data-gv-theme="light" data-gv-density="comfortable">
```

    <ul className="gv-doc-list">
      <li>`data-gv-theme`: `light` or `dark`</li>
      <li>`data-gv-density`: `compact`, `comfortable`, or `spacious`</li>
    </ul>
  </div>

  ## 4. Import standalone components where you use them

  <div className="gv-doc-panel">
```ts
import { Component } from '@angular/core';
import { Button, InputComponent, Tabs } from 'gravitas-ui';

@Component({
  standalone: true,
  selector: 'app-account-settings',
  imports: [Button, InputComponent, Tabs],
  template: `
    <gv-input label="Account name" placeholder="Acme Corp" />

    <gv-button variant="primary">Save changes</gv-button>
  `,
})
export class AccountSettingsComponent {}
```
  </div>

  ## What not to do

  <div className="gv-doc-grid gv-doc-grid--wide">
    <div className="gv-doc-card">
      <h3 className="gv-doc-card-title">Avoid</h3>
      <ul className="gv-doc-list">
        <li>loading Bootstrap and Gravitas multiple times</li>
        <li>overriding internal Gravitas classes ad hoc in each app</li>
        <li>mixing incompatible Bootstrap versions across projects</li>
        <li>rebuilding existing Gravitas primitives with custom local markup</li>
      </ul>
    </div>

    <div className="gv-doc-card">
      <h3 className="gv-doc-card-title">Prefer</h3>
      <ul className="gv-doc-list">
        <li>one shared app-shell setup across company projects</li>
        <li>token overrides for branding and theming</li>
        <li>documented component usage through Storybook</li>
        <li>feeding missing patterns back into Gravitas instead of forking them locally</li>
      </ul>
    </div>
